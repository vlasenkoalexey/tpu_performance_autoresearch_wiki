# Qwen3 JAX current v124 - 2k batch16 Splash BQ1024 BKV2048 reduce-scatter latency 4

## Hypothesis under test

**Hypothesis**: v095 established forward `SPLASH_BKV=2048` as useful at 2k, but
the forward query tile remains at `SPLASH_BQ=2048`. Related Llama kernel tuning
favored `block_q=1024` in similar Splash shapes. Keeping the successful
full-sequence KV tile while reducing the query tile may improve the forward
custom-call bucket or memory scheduling.

**Mechanism**: Reuse the v114/v119 graph settings but add:

```bash
export SPLASH_BQ=1024
```

Keep `SPLASH_BKV=2048`, `SPLASH_BKV_COMPUTE=2048`, DKV tiles at 2048, VMEM
98304, shmem100, reduce-scatter latency multiplier 4, all-reduce latency
multiplier 2, SparseCore all-gather offload disabled, aggregator enabled,
batch 16, and sequence 2048 fixed.

## Verdict

Refuted. `SPLASH_BQ=1024` is graph-active but worsens the HLO and profile:
copy/fusion counts rise, Tokamax appears twice, step time regresses to the
3731 ms band, and throughput falls below the frontier. Keep the v114/v119
Splash shape: `SPLASH_BQ=2048`, `SPLASH_BKV=2048`.
