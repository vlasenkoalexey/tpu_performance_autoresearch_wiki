# Qwen3 JAX current v129 - 2k batch16 Splash BKV2048 reduce-scatter latency 4 DKV compute 512

## Hypothesis under test

**Hypothesis**: The v119 frontier still spends a material fraction of profile
time in Tokamax Splash backward custom calls. Reducing only the backward KV
compute tile can improve that bucket while keeping the forward Splash tiling and
collective schedule fixed.

**Mechanism**: Start from the v119 named 2k frontier and add only:

```bash
SPLASH_BKV_DKV_COMPUTE=512
```

Keep batch 16, sequence 2048, forward Splash BQ/BKV/BKV_COMPUTE 2048,
backward BQ_DKV/BKV_DKV 2048, VMEM 98304, shmem100, scheduler rerun 4, async
collective fusion enabled, all-gather and reduce-scatter ACF enabled,
all-reduce ACF disabled, RS latency multiplier 4, AR latency multiplier 2,
SparseCore all-gather offload disabled, reduce-scatter/all-reduce offload
enabled, and aggregator enabled.

## Verdict

Refuted. The DKV compute tile change perturbed the optimized HLO hash but did
not reduce memory, op counts, or profile step time, and throughput remained
below the v119 frontier. Do not carry `SPLASH_BKV_DKV_COMPUTE=512`.
