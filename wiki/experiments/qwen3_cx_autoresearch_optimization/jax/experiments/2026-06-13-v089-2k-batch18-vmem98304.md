# Qwen3 JAX current v089 - 2k batch18 VMEM 98304

## Hypothesis under test

**Hypothesis**: v086 is the current 2k frontier at per-chip **batch_size=16**,
**70,564 tok/s**, and **45.4% script MFU**, with about **1.99 GiB** runtime HBM
free. A smaller batch step to **18** may still fit and improve MFU by increasing
matmul work per step.

**Mechanism**: Reuse v086 exactly, changing only `--batch_size=18`. Keep
`--seqlen=2048`, scoped VMEM **98304 KiB**, latency-hiding scheduler rerun
**3**, no-scan remat, activation sharding, Tokamax Splash, MaxText CE,
SparseCore reduce-scatter/all-reduce offload, SparseCore collective aggregator,
and shared-memory limit **90**.

**Predicted signal**: Support requires clean finite loss plus throughput above
v086's **70,564 tok/s** and no severe XProf variance or memory pressure. Compile
OOM, runtime OOM, or a flat/slower result closes raw batch scaling above
batch16 for this 2k frontier.
