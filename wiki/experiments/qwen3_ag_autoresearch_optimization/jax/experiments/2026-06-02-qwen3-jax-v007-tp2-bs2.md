# Experiment: v007 Tensor Parallelism (TP=2)

**Status:** Completed
**Lane:** jax
**Date:** 2026-06-02

## Hypothesis under test
- **Hypothesis**: Tensor parallelism (TP=2, FSDP=4) halves the per-chip activation memory footprint for the attention layer by distributing the sequence representations and KV heads across chips. This structural distribution will bridge the 2.36 GB HBM gap that caused the baseline to OOM at batch size 2, unblocking batch scaling without kernel-level fused attention.
- **Mechanism**: Use the baseline image and set `--tp_parallelism=2` and `--batch_size=2` in the launch command.
- **Predicted signal**: The workload will compile and run successfully at `bs=2` without OOM, achieving >22% MFU.
- **Falsification criterion**: The workload still OOMs, or the added cross-device communication overhead from TP (`collective-permute` for `all-reduce` after `o_proj` and MLP) degrades MFU below the `bs=1` baseline (20.0%).

## Configuration
- **Model**: Qwen/Qwen3-8B
- **Framework**: JAX (Flax NNX)
- **Batch Size**: 2
- **Seq Length**: 2048
- **Remat Policy**: `checkpoint_dots_with_no_batch_dims`
- **Attention**: SDPA (Baseline)
- **Image**: `<your-registry>/torchtitan-images/qwen3-8b-jax:latest`
- **Flags**: `--tp_parallelism=2 --batch_size=2`

## Setup
Launched directly via `gke-cluster-runner` using the baseline image with the modified flags.

## Results
- **Status**: Crashed
- **Error**: `XLA ResourceExhaustedError (OOM) trying to allocate 256.00M HLO temp buffers.`

## Verdict
**Refuted**.
Tensor parallelism successfully distributed some intermediate activations, but the baseline SDPA attention still materialized the `[batch, num_heads, seq_len, seq_len]` `[2, 16, 2048, 2048]` attention score matrix (256 MB per chip) during the JVP backward pass, which still exceeded the total HBM limits.

## Next Steps
Relying on standard SDPA with TP=2 is insufficient to scale up batch size because the `[seq, seq]` matrix scales quadratically and causes XLA memory pressure. We need to rely on the `splash_attention` implementation (running in parallel as `v008`) or Tokamax CE (`v006`) to structurally eliminate these massive intermediate buffers.
