# Qwen3-8B jax v021 — splash symmetric blocks bq=bkv=1024 (frontier)

## Hypothesis under test

**Hypothesis**: The lane's **upstream** splash currently runs asymmetric forward blocks
`bq=2048, bkv=1024`. `splash_attn.py` documents symmetric `block_q == block_kv == 1024` as the
torchax-autotuned "universal winner". Setting `SPLASH_BQ=1024` (→ symmetric 1024/1024 forward) on
the v018 frontier lifts MFU above 35.8% by improving the attention kernel's MXU tile efficiency /
reducing VMEM pressure.

**Mechanism**: v018 frontier config (`--use_remat --use_splash --batch_size=4 --seqlen=2048` + 7
LIBTPU scheduler flags + scoped_vmem, **upstream** splash i.e. `USE_TOKAMAX_SPLASH` unset) + env
`SPLASH_BQ=1024` (bkv already 1024; dkv blocks unchanged at 2048). Flag-only, no rebuild.

**Predicted signal**: MFU > 35.8% (target +0.5–1 pp); splash custom-call time-share drops; loss
parity (block sizing is numerically exact); exit 0.

**Falsification criterion**: MFU ≤ 35.8% beyond noise (2048 was already optimal for qwen3), or a
scoped-VMEM OOM.

## Setup

- **Hardware**: v6e-8 (fsdp=8), image `qwen3-8b-jax:v006-splash`. Flag-only (env).
- **Launch**: v018 frontier (upstream splash, `--use_remat --use_splash --batch_size=4 --seqlen=2048` + 7 LIBTPU scheduler flags + scoped_vmem) **+ env `SPLASH_BQ=1024`**. No tokamax, no jf_spmd.
- **Blocks confirmed** (all 28 layers): `bq=1024 bkv=1024 bq_dkv=2048 bkv_dkv=2048 fused_bwd=True`.

## Verdict

**Refuted** (for the seq2048 sub-case). Symmetric `bq=bkv=1024` is −3.3 pp / −9.4% vs the asymmetric `bq=2048, bkv=1024` frontier — the torchax-autotuned "symmetric-1024 universal winner" does **not** transfer to qwen3-jax. The signal is clear: **larger `bq` is better** (2048 = seqlen, maximal forward q-block, wins). At seq2048 `bq` is already maxed, so forward-block tuning is exhausted here. **The actionable follow-up is at seq8192**, where `bq=2048` is only ¼ of seqlen — growing the forward block there (the program-target seq) is untested and well-motivated by this monotone trend. Frontier stays v018.
