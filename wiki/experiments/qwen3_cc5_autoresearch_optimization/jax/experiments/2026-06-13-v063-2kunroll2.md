## Hypothesis under test

[qwen3-jax-2k-scan-unroll2](../../../../hypotheses/qwen3-jax-2k-scan-unroll2.md).

**Mechanism**: 2-layer scan body → the latency-hiding scheduler can overlap
layer N's grad-RS under layer N+1's backward compute. The XLA S(5)
unroll×offload bug does not apply (no host offload in this recipe — context
in HBM).

**Predicted signal**: ARS bucket 158 → ≤ 60 ms; scan `while` trip count
18 (36/2); step ≤ 1,262 ms (PASS = ≥ +3%, tok/s/chip ≥ 8,107); loss exact
(12.0960/12.0969/12.0933 — graph-shape-only change). OOM at compile ⇒
measure margin, re-arm at bs4. Flat ⇒ the scheduler doesn't exploit the
window (mirrors the 8k AG-overlap refutation lineage) — unroll closes.

## Verdict

**refuted** — unroll=2 is a clean −4.1% regression; ARS bucket FLAT
(157.3 vs 157.8 ms — zero overlap despite the window existing in the
graph); the regression is pure body overhead (loop fusion +34.9, state
re-packing +19.7). Unroll=3/4 strictly dominated (~linear overhead growth,
same zero benefit). Direction closed; hypothesis retired. Side-findings:
(a) the XLA S(5) unroll bug is confirmed **offload-specific**; (b) the
exposed-ARS target (157.8 ms = 12.1% of step) is reachable ONLY via manual
async pipelining (explicit RS-start/done with interleaved compute,
collective-matmul-style) — L effort, low prior given the 8k
manual-prefetch refutation lineage.
