# Qwen3-8B jax v026 — MaxText flag completion @ bs3/seq8192

A/B against [v025](2026-06-12-v025-offload-scan-bs3.md): identical model
config (scan + offload_attn + splash + chunked CE f32-x, bs3 @ seq 8192);
the only diff is the LIBTPU flag stack, completed to MaxText-recipe parity
(minus the 2 compile-fatal SparseCore flags).

## Hypothesis under test

[qwen3-jax-maxtext-flag-completion](../../../../hypotheses/qwen3-jax-maxtext-flag-completion.md):
the 13 unadopted recipe flags (windowed-einsum disable +
async-collective-fusion family + RS layout + DP overlap + vmem 98304) are
worth ≥ +2% tok/s/chip at this shape. v025's profile shows windowed-einsum
collective matmul ACTIVE in our compile; MaxText disables it.

**Flags added vs v025** (from the mt-v001 26-flag stack):
`--xla_tpu_scoped_vmem_limit_kib=98304` (was 81920),
`--xla_tpu_use_minor_sharding_for_major_trivial_input=true`,
`--xla_tpu_relayout_group_size_threshold_for_reduce_scatter=1`,
`--xla_tpu_assign_all_reduce_scatter_layout=true`,
`--xla_tpu_enable_data_parallel_all_reduce_opt=true`,
`--xla_tpu_data_parallel_opt_different_sized_ops=true`,
`--xla_tpu_enable_async_collective_fusion=true`,
`--xla_tpu_enable_async_collective_fusion_fuse_all_gather=true`,
`--xla_tpu_enable_async_collective_fusion_multiple_steps=true`,
`--xla_tpu_overlap_compute_collective_tc=true`,
`--xla_enable_async_all_gather=true`,
`--xla_tpu_enable_async_collective_fusion_fuse_all_reduce=false`,
`--xla_jf_spmd_threshold_for_windowed_einsum_mib=1000000`.
Excluded: `--xla_tpu_enable_sparse_core_collective_offload_all_reduce`,
`--xla_tpu_enable_all_reduce_offload_tracing` (RET_CHECK crash, mt-v001
attempt 1).

**Predicted signal**: single phase bs3 @ 8192, 20 steps; PASS if tok/s/chip
> 5,489 (v025 + 2%); the MaxText-match bar stays 6,953; v019 frontier bar
5,898. Loss must match v025's bs3 series (same seed/shape).

**Falsification criterion**: within ±1% of v025's 5,381 → refuted.

## Setup

- Hardware: v6e-8, fsdp=8, tp=1, 1 slice of `alekseyv-tpu-v6e8-spot-xpk`.
- Image: `qwen3-8b-jax:v6e8-qwen3-8b-jax-20260612-v024-offload` (reused).
- Workload: `alekseyv-qwen3-cc5-jax-v026-mtfl`.
- Single phase: bs3 seq8192, scan + offload_attn + splash + chunked CE
  f32-x, 20 steps, profiled 12–14, HLO dump.

## Results

20 steps clean, exit 0, no OOM, no NaN. Loss series matches v025
bit-or-near (step 0/2/3 exact, step 1 Δ0.0002) — the flags do not change
numerics. Compile cost: step 0 = 51.6 s + step 1 = 44.4 s (new flag stack
fully invalidated the persistent cache + tokamax re-autotune); steady state
from step 2. Step spread across the window: 8.5 ms (~0.2%).

| Metric | v025 (old 12-flag stack) | **v026 (completed 25-flag stack)** | v019 frontier (bs1) | MaxText (bs3) |
|--------|--------------------------|-------------------------------------|---------------------|----------------|
| Step time (median, steps 2–11+15–19) | 4,566.6 ms | **4,142.4 ms (−9.3%)** | 1,389.0 ms (65,536 tok) | 3,535 ms |
| tok/s/chip | 5,381 | **5,933 (+10.3%)** | 5,898 | 6,953 |
| MFU (our accounting) | 30.9% | **34.0%** | 33.6% | ≈39.6% |
| vs frontier bar 5,898 | −8.8% | **+0.6% — CLEARS** | — | — |
| vs MaxText 6,953 | −22.6% | **−14.7%** | −15.2% | — |

All three bars resolved: PASS (> 5,489) ✓, **frontier (> 5,898) ✓** —
first time the bs3 route beats the bs1 frontier — MaxText (6,953) not yet.

Attribution of the −424 ms: windowed-einsum ring elimination (−341 ms loop
fusion — the scan body no longer carries per-iteration collective-permutes),
RS-layout flags (−90 ms data formatting), leaner matmul scheduling
(−126 ms conv fusion); partially offset by +168 ms of now-explicit
collective accounting (RS fusion, async-done).

- **MXU utilization 53.4%** (v025: 48.5%) — compute-bound.
- **HBM peak 19.66 / 31.25 GiB (62.9%)** — stack 13.83 + heap 5.83,
  **11.59 GiB free** (v025: 27.34 peak — layout flags also shrank the
  live-set). bs4 projects ≈ 30.1 GiB — marginal, probe-worthy.

## HLO Dump

- **GCS**: `gs://tpu-pytorch-alekseyv-us-central2/autoresearch/qwen3_cc5/2026-06-12-qwen3-jax-v026-mtfl/hlo/`
  (main module `module_0379.jit_train_step`, 5,689 instructions, scheduled).
- **All three predicted flag signatures verified**: (a) `collective-permute`
  count = **0** — the windowed-einsum/collective-matmul ring is gone;
  (b) `AsyncCollectiveStart/Done` pairs wrap the FSDP all-gathers;
  (c) explicit `all-reduce-scatter` computations with custom barriers.
- offload_attn path intact: 529 copy-done ops chained
  slice-done → ConcatBitcast into splash operands; fully pipelined.
- Note: vmem 98304 likely silently capped (v6e ≤ 65472 KiB); no HLO-visible
  effect — the win is attributable to the windowed-einsum + layout +
  collective-fusion flags, not vmem.

**Bottleneck verdict (profile-analyzer)**: largest recoverable cost is now
the **splash attention bucket — 1,088 ms/step (26.3%)**, 6 kernel
invocations per scan iteration, untouched by the flag stack. Flag tuning is
unlikely to close the remaining 607 ms to MaxText; the attention kernel
config (block sizes / fused bwd / GQA mapping) is the actionable question.

## Verdict

**supported — FRONTIER SHIFT.** 5,933 tok/s/chip @ bs3/seq8192 beats the
v019 frontier (5,898, bs1) by +0.6% with step-time spread ~0.2%; numerics
unchanged (loss matches v025 ±0.0002, and v025's bs1 phase matched the
seed-0 series exactly). Hypothesis confirmed structurally: the
windowed-einsum disable is the dominant lever (−341 ms), with RS-layout and
collective-fusion flags contributing. The flag stack is now **lane-standard
for seq8192** (recorded in the model page How-to-run). Bonus finding: under
the completed flags, host-offload DMA becomes fully hidden — the offload
route flips from throughput-negative (v025) to the frontier config.
