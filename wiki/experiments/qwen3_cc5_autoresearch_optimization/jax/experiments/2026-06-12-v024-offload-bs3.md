# Qwen3-8B jax v024 — host-offload remat + bs3 @ seq 8192

Ports the MaxText recipe's memory mechanism
([mt-v001 gap analysis](../../maxtext/experiments/2026-06-12-mt-v001-base.md))
into our trainer: offload the remat-saved attention tensors to pinned host
memory, then take the batch step MaxText takes.

## Hypothesis under test

**Hypothesis**: `offload_attn` remat (q/k/v/attn_out → pinned host) + the
HOST_OFFLOAD scheduler flags lets bs3 @ seq 8192 fit and exceed 5,898
tok/s/chip — closing toward MaxText's 6,953 on the identical architecture.

**Mechanism**: `_remat_policy("offload_attn")` =
`save_and_offload_only_these_names(saved=[], offloaded=[attn_q,attn_k,attn_v,
attn_out], device→pinned_host)`; FFN/norms rematerialized exactly as in
save_attn (v015-validated numerics path). Offloaded mass at bs3 ≈ 9 GiB
leaves HBM for the batch. HOST_OFFLOAD scheduler flags included as part of
the mechanism (v010 showed they're inert without offload traffic).

**Predicted signal**: phase 1 (bs1) — step time near 1389–1476 ms band with
host-transfer activity visible (no NaN; loss = the known seed-0 series);
phase 2 (bs3) — fits (< 31 GiB), tok/s/chip > 5,898; stretch ≥ 6,500.

**Falsification criterion**: bs3 OOM, or bs3 tok/s/chip ≤ 5,898 + 1%
(transfer-stall bound) → refuted.

## Setup

- Hardware: v6e-8, fsdp=8, tp=1, 1 slice of `<your-cluster>`.
- Image: `qwen3-8b-jax:v6e8-qwen3-8b-jax-20260612-v024-offload`.
- Workload: `alekseyv-qwen3-cc5-jax-v024-off3`.
- Phases (both seq8192, splash + chunked CE f32-x + offload_attn remat +
  vmem flag + HOST_OFFLOAD scheduler flags; no scan, no SKIP_EVERY):
  ph1 = bs1 sanity/overhead (12 steps); ph2 = bs3 (20 steps, profiled).

## Results

**Phase 1 (bs1 + offload_attn) — PASS.** 12 steps clean, loss series matches
the seed-0 reference exactly (12.0980 / 12.0984 / 12.0942 …), no NaN.

| Metric | v015 save_attn (bs1 ref) | ph1 offload_attn (bs1) | delta |
|--------|--------------------------|-------------------------|-------|
| Step time (steady, steps 2–3 + 6–11) | 1475.7 ms | ~1832 ms | **+24%** |
| tok/s/chip | 5,551 | 4,475 | −19% |
| MFU (our accounting) | 31.8% | 24.5% | −7.3 pp |

At bs1 the host transfers are NOT latency-hidden — offload costs +24% step
time over saving the same tensors in HBM. (Expected: the point of offload is
to buy the batch, not bs1 speed.)

**Phase 2 (bs3 + offload_attn) — compile-time OOM.**
`RESOURCE_EXHAUSTED: Ran out of memory in memory space hbm. Used 59.41G of
31.25G hbm. Exceeded hbm capacity by 28.16G.` Never reached step 0.

## Profile

Not captured — phase 2 OOM'd at compile, before `profile_start_step=12`;

## HLO Dump

`gs://<your-bucket>/autoresearch/qwen3_cc5/2026-06-12-qwen3-jax-v024-off3/hlo/`
— `module_0092.jit_train_step` (phase-2 bs3) incl. `oom_intermediate_module`
+ `oom_top_memory_consuming_instructions`.

Key findings (the offload **mechanism worked**; the **unrolled backward
liveness** is what OOM'd):

1. **Annotations emitted correctly**: `before_optimizations` has 144
   `annotate_device_placement(pinned_host)` custom-calls = 36 layers × 4
   tensors, all **bf16** at global shapes (e.g. `bf16[24,32,8192,128]`).
2. **Host-offloader pass engaged**: the OOM-time module has **432 `S(5)`
   (host memory space) references and 0 leftover annotations** — the saved
   q/k/v/attn_out really do live on the host.
3. **What OOM'd — HLO temps, 59.40 G total**:
   - 2.34 G `f32[4096,153600]` broadcast in tokamax CE backward (the known
     f32 dw accumulator, v008b) + 1.17 G CE-fwd pad;
   - **≥18 × 384 MB `f32[3,8192,32,128]` copies** (top-20 report truncated;
     the bulk of 59 G is this pattern) from
     `checkpoint/rematted_computation/convert_element_type` — the f32
     rematerialized intermediates (QK-norm/RoPE recompute) of the
     **backward** pass, held live concurrently across the 36 **unrolled**
     layers instead of streaming layer-by-layer.

The scheduler-memory-pressure flags (`scheduler_memory_pressure_tracking`,
`latency_hiding_scheduler_rerun=2`) did not bound this. MaxText avoids it
structurally: its decoder is a **scan over layers**, which hard-bounds
backward liveness to ~one layer's working set per iteration. We have scan
merged as opt-in ([v020](2026-06-12-v020-scan.md), −8.0 GiB live-set) — the
v024 image already contains both code paths, so the retry is launch-only.

## Verdict

**refuted** (as-configured: offload_attn on the **unrolled** 36-layer loop
does not unlock bs3 @ seq 8192 — falsification criterion "bs3 OOM" met).

Scope note: this refutes the *unrolled* configuration, not the offload
mechanism itself — numerics are clean (ph1 loss parity exact), the tensors
demonstrably move to host (432 S(5) refs), and the OOM mass is rematted
backward temps whose concurrency is an artifact of the unrolled loop. The
parent hypothesis [qwen3-jax-host-offload-remat](../../../../hypotheses/qwen3-jax-host-offload-remat.md)
stays `in_progress`; the structural fix (scan) is dispatched as v025.
