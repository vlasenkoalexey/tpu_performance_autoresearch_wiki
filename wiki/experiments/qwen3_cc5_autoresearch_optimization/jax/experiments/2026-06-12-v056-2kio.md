# Qwen3-8B jax v056 — io weight layout at 2k

Second lever of the
[2k frontier port](../../../../hypotheses/qwen3-jax-2k-frontier-port.md),
independent of the flag/substrate question: `JAX_WEIGHT_LAYOUT=io` on the
plain [v052](2026-06-12-v052-2kbase.md) recipe (unrolled, vmem-only flags).
Single axis: only the env var changes.

## Hypothesis under test

**Mechanism**: (in,out)-stored Linear kernels make every decoder x@W
transpose-free (+2.8% at 8k, [v050](2026-06-12-v050-weight-io.md)/v051 —
the conv-fusion BW-utilization lever). Shape-independent in principle; conv
fusion is 37.2% of the 2k step. lm_head auto-excluded (tokamax-CE contract;
CE here is plain XLA, but the exclusion is harmless).

**Predicted signal**: conv-fusion bucket shrinks; step < 667.6 ms. PASS =
tok/s/chip ≥ +1.5% (step ≤ ~657.6 ms — 8k gave +2.8%; bar set at half).
Loss parity 12.0914/12.0986/12.0947 ±0.001 (v050 CPU smoke was
bit-identical; seed-draw parity built into the io path).

**Falsification**: step within noise of 667.6 ms ⇒ the layout win is
shape-gated (e.g. the transpose elision only matters at 8k's tile shapes) —
io closes at 2k.

## Setup

- Cluster charles-v6e (1 slice), image `latest` (digest 3ab5bcf1), workload
  `alekseyv-qwen3-cc5-jax-v056-2kio`.
- Command = v052's + `export JAX_WEIGHT_LAYOUT=io`:
  ```bash
  export LIBTPU_INIT_ARGS="--xla_tpu_scoped_vmem_limit_kib=81920"
  export JAX_WEIGHT_LAYOUT=io
  python -u train.py --model_id=Qwen/Qwen3-8B --use_real_data=False \
    --batch_size=2 --seqlen=2048 --train_steps=20 --tp_parallelism=1 \
    --weights_dtype=bf16 --use_splash=True \
    --profile_dir=<gcs_root> --profile_start_step=12 --profile_steps=3
  ```
- GCS root: `gs://tpu-pytorch-alekseyv-us-central2/autoresearch/qwen3_cc5/2026-06-12-qwen3-jax-v056-2kio/`

## Results

20 steps clean (exit 0). Compile ~40 min fresh (the env var changes the
traced graph → compile-cache miss — itself evidence the layout flip reached
the program).

| Metric | v052 baseline | **v056 (io layout)** | Δ |
|--------|---------------|----------------------|---|
| Step (median steady, ex profile steps) | 667.6 ms | **666.4 ms** | −1.2 ms (−0.18%) |
| Throughput | 49,090 tok/s | 49,185 tok/s | +0.19% |
| Loss (steps 0/1/2) | 12.0914 / 12.0986 / 12.0947 | 12.0914 / 12.0986 / 12.0947 | exact ✓ |

Within the ~1–2 ms run-to-run noise band. PASS bar (≤657.6 ms) not met.

## HLO Dump

- **GCS**: `gs://tpu-pytorch-alekseyv-us-central2/autoresearch/qwen3_cc5/2026-06-12-qwen3-jax-v056-2kio/hlo/`
  (187 files; `module_0112.jit_train_step`, 24.25 MiB after_opt — ~4,500

- **Hypothesis-firing audit: CONFIRMED — refuted verdict is sound.**
  Before-opt HLO shows the flip: `k_proj bf16[4096,1024]` (vs v052
  `[1024,4096]`), `gate_proj bf16[4096,12288]` (flipped), `down_proj
  bf16[12288,4096]` (flipped); dot `rhs_contracting_dims={1}→{0}`; no bridge
  transposes inserted; **lm_head `bf16[151936,4096]` unchanged** (excluded
  as designed). After-opt: −175 transpose ops, −147 copy ops vs v052. The
  40-min fresh compile (cache miss) is corroborating structural proof.
- Conclusion: genuine **shape-gated null** — transpose-elision is
  tile-shape-dependent; at [2,2048,·] activations conv-fusion time is
  already near-optimal.

## Verdict

**refuted** — the (in,out) weight layout is performance-neutral at 2k/bs2
(−0.18%, within noise); the seq-8192 win (+2.8%, v050/v051) is
**shape-gated**: at [2,2048,·] activations the transpose elision that paid
at 8k doesn't move the BW-utilization needle. Loss exactly equal to v052 —
numerics-preserving as designed. Knobs matrix updated: io layout = per-shape
knob (ON at 8192, neutral at 2048).
