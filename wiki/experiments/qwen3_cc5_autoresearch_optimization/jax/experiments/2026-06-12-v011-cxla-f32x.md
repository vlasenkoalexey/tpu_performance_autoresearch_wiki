# Qwen3-8B jax v011 — chunked_xla CE, f32-x boundary (+ splash)

Tests [qwen3-jax-chunked-xla-ce-f32x](../../../../hypotheses/qwen3-jax-chunked-xla-ce-f32x.md):
the corrective derivative of [v008b](2026-06-12-v008-chunked-xla-ce.md).

## Hypothesis under test

**Hypothesis**: f32-x-only chunked_xla CE restores loss parity with the
plain-CE reference (within bf16 noise) at ≤ +5% step time vs plain CE,
establishing the validated streamed-CE building block for seq 8192.

**Mechanism**: `h_flat.astype(jnp.float32)` (32 MB at bs1) before the
shard_map'd `chunked_xla` call; w stays bf16. The kernel's lse dtype follows
`x.dtype` ⇒ f32 accumulation. CPU smoke confirms f32 loss output.

**Predicted signal**: phase-2 losses smooth f32 values matching phase 1
within ≤ 0.01 (vs v008b's quantized −0.73 offset); zero `[2048,151936]`
buffers; step ≈ 467 ms (same as v008b).

**Falsification criterion**: loss offset beyond bf16 noise → invalid; step
time > plain-CE + 5% → refuted for the seq-8192 path.

## Setup

- Hardware: v6e-8, fsdp=8, tp=1, 1 slice of `alekseyv-tpu-v6e8-spot-xpk`.
- Image: `qwen3-8b-jax:v6e8-qwen3-8b-jax-20260612-v011-cxla-f32x`
  (FROM v008b image — dev tokamax — with the f32-x train.py).
- Workload: `alekseyv-qwen3-cc5-jax-v011-cxf32`.
- In-workload A/B (seed 0), both phases splash-on: phase 1 plain CE 8 steps;
  phase 2 `--use_tokamax_ce=True --tokamax_ce_impl=chunked_xla` 20 steps,
  profiled.

## Results

Both phases exit 0. **Decisive parity check PASSED**: phase-2 losses are
smooth f32 values tracking phase 1 within **≤ 0.007** on shared steps (v008b's
quantized −0.73 pattern absent). Autotune key confirms the boundary:
`{"x":"f32[2048,4096]","w":"bf16[4096,151936]"}`.

| Metric | Phase 1 (plain CE + splash) | Phase 2 (chunked_xla f32-x + splash) | Δ |
|--------|------------------------------|---------------------------------------|---|
| Steady step time | 453.0 ms | 467.7 ms | **+3.2%** |
| Throughput | 36,139 tok/s | 35,030 tok/s (median) | −3.1% |
| Loss parity | reference | ≤ 0.007 on all shared steps | ✓ |

## Verdict

**supported** (as the parity-enabler hypothesis): loss parity within bf16
noise AND step-time overhead +3.2% ≤ the +5% bound. chunked_xla CE with
f32-x/bf16-w is now the lane's validated streamed-CE recipe. NOT merged to
trunk as default (it costs +3.2% at seq2048 where it saves no memory — see
v008b's dw-accumulator analysis); it is deployed where it pays:
**seq 8192**, probed next (v013).
