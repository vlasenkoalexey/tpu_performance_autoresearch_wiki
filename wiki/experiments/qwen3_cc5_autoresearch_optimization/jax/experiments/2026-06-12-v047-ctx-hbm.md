# Qwen3-8B jax v047 — context in HBM

Launch-only refinement of the frontier: `--remat_policy=save_qkv_ctx_hbm`
keeps the splash context residuals (4.57 GiB stacked) in HBM instead of
host — peak ≈ 22.03 + 4.6 ≈ 26.6 GiB, fits — eliminating the +25 ms DMA
residual the v044 profile measured.

## Hypothesis under test

[qwen3-jax-ctx-hbm-only](../../../../hypotheses/qwen3-jax-ctx-hbm-only.md):
PASS > 6,782 tok/s/chip (frontier +1%; step < 2,413 ms). Loss must match
12.1029/12.0996/12.0944 ±0.001 (policy-only).

**Diff vs v045**: `--remat_policy=save_qkv_ctx_hbm` (vs `save_qkv_ctx`).

## Setup

- Image: `latest` (= v044-ctx digest 5acf9bbb); workload
  `alekseyv-qwen3-cc5-jax-v047-ctxh`; 20 steps, profiled 12–14, HLO.

## Results

20 steps clean, no OOM, loss exact (12.1029 / 12.0995 / 12.0944).

| Metric | v045 frontier (ctx offloaded) | **v047 (ctx in HBM)** |
|--------|-------------------------------|------------------------|
| Step (median 2–11+15–19) | 2,436.9 ms | **2,416.5 ms (−20.4 ms)** |
| tok/s/chip | 6,715 | **6,780 (+1.0%)** |
| MFU | ≈38.6% | **≈38.9%** |
| vs strict PASS 6,782 | — | −2 tok/s/chip (0.03%, rounding) |
| vs MaxText 6,953 | −3.4% | **−2.5%** |

The +25 ms DMA residual measured in v044's profile is recovered (minus
~5 ms of HBM-pressure effects). HBM fits (~26.6/31.25 projected).
