# Qwen3-8B jax v002 — splash attention

Tests [qwen3-jax-splash-attention](../../../../hypotheses/qwen3-jax-splash-attention.md)
on `8B/v6e-8`: replace the XLA-SDPA einsum attention with the GQA-native splash
Pallas kernel, against the [v000 baseline](2026-06-02-qwen3-jax-v6e8-baseline.md)
at seq 2048 bs 1.

## Hypothesis under test

**Hypothesis**: Replacing XLA-SDPA (`jnp.einsum` + full `[B,H,S,S]` softmax)
with the splash attention Pallas kernel reduces step time at seq 2048 beyond
noise (and structurally unlocks seq 8192 later), because splash tiles the
attention and never materializes the S×S score matrix in HBM.

**Mechanism**: Env-gated dispatch in `model/modeling_qwen3.py`
(`JAX_ATTENTION_IMPL=splash` → `_attn_splash`), `splash_attn.py` copied from
the llama3 jax lane (shard_map wrapper, `P('fsdp','tp',None,None)`, autotuned
blocks bq=2048 bkv=1024, fused bwd). Two corrections vs the llama3 dispatch:
q is pre-scaled by `1/sqrt(head_dim)` (upstream splash applies no scaling) and
the kernel output `(B,Hq,T,D)` is transposed to `(B,T,Hq,D)` before `o_proj`
(upstream returns heads-major; the llama3 lane reshapes without transposing —
flagged for a separate observation). QK-norm runs before RoPE, outside the
kernel — unchanged.

**Predicted signal**: a splash `custom-call` appears in the op profile
(`custom-call` bucket grows from ~0%); the SDPA `[B,H,S,S]` softmax disappears
from `loop fusion`/`convolution fusion` attribution; train_step HLO module
contains `custom_call` targeting the Mosaic splash kernel. Step time at
seq 2048 expected flat-to-modestly-better (attention is a minor share at S=2048,
bs1); the decisive signal is kernel-fired + loss parity.

**Falsification criterion**: no step-time improvement at seq 2048 AND still
OOMs at seq 8192 → refuted. Loss trajectory diverging from the in-run XLA
reference beyond bf16 noise → invalid (layout/scaling port bug).

## Setup

- Hardware: v6e-8 (2 hosts × 4 chips), fsdp=8, tp=1, 1 slice of
  `<your-cluster>`.
- Image: `qwen3-8b-jax:v6e8-qwen3-8b-jax-20260612-v002-splash` (fork commit `3747321`).
- Workload: `alekseyv-qwen3-cc5-jax-v002-splash`.
- In-workload semantics A/B (deterministic data, seed 0): phase 1 runs 8 steps
  with XLA SDPA (reference), phase 2 runs 20 steps with `--use_splash=True`
  (measured + profiled). First 8 losses must match within bf16 noise.
- Command:
  ```bash
  python -u train.py --batch_size=1 --seqlen=2048 --train_steps=8 \
      --use_splash=False   # phase 1: loss reference
  python -u train.py --batch_size=1 --seqlen=2048 --train_steps=20 \
      --use_splash=True --profile_dir=<gcs_root> --profile_start_step=12 --profile_steps=3
  ```

## Results

Both phases exit 0; wall time 7m10s. Steady-state = phase-2 steps 2–19
excluding profiler/GCS-flush spikes (steps 12, 17).

| Metric | Phase 1 (XLA SDPA) | Phase 2 (splash) | Δ |
|--------|--------------------|------------------|---|
| Steady step time | 512 ms | **453 ms** | **−11.5%** |
| Throughput | 31,981 tok/s (3,998/chip) | **34,705 tok/s (4,338/chip)** | **+8.5%** |
| MFU (trainer) | 20.6% | **22.3%** | **+1.7 pp** |
| Kernel fired | — | `[splash] bq=2048 bkv=1024 … fused_bwd=True` ×36 | ✓ |

**Loss parity (semantics check)**: first 8 steps, phase 1 vs phase 2 on
identical data (seed 0): max |Δ| = 0.0005 (step 0 bit-identical 12.0933) — all
8 within bf16 noise → splash is numerically equivalent here, including the
pre-scale + output-transpose corrections.

Phase-1 reference also independently reproduces the v000 baseline
(512 ms / 31,981 ≈ 31,955 tok/s / 20.6 ≈ 20.5% MFU) — run-to-run noise < 0.2%.

## Verdict

**supported** — (a) −11.5% step time / +1.7 pp MFU, far beyond the <0.2%
run-to-run noise; (b) loss parity within bf16 noise across all 8 reference
steps (semantics preserved); (c) no tracked-metric regression (HBM strictly
lower — score transients eliminated). **Frontier shifts to 453 ms / 34,705
TPS / 22.3% MFU.** Fork merged to trunk; stable image rebuilt.

## Observations

- The llama3 jax lane's `_attn_splash` neither pre-scales q nor transposes the
  splash output `(B,Hq,T,D)→(B,T,Hq,D)`; upstream splash requires both
  (no internal 1/√d; heads-major return). Qwen3's loss-parity A/B proves the
  corrected port is the equivalent one. Worth an audit on the llama3 lane —
  filed as [observation](../../../../observations/llama3-jax-splash-scaling-transpose-audit.md).
- Collective share is now the top non-matmul bucket (25.1% + 4.5%) — the
  next profile-driven lever after memory unlocks.
