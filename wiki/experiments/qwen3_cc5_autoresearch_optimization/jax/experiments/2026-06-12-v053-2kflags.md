# Qwen3-8B jax v053 — 21-flag stack at 2k

First lever of the
[2k frontier port](../../../../hypotheses/qwen3-jax-2k-frontier-port.md):
the lane-standard 21-flag LIBTPU stack (v026/v035 lineage, +10.3% at
seq8192), applied to the [v052](2026-06-12-v052-2kbase.md) baseline. Single
axis: only `LIBTPU_INIT_ARGS` changes.

## Hypothesis under test

**Mechanism**: the v052/v007 profile shows ~30% collectives (CP-done 15.7% +
AG 14.3%) — the RS-layout / windowed-einsum-disable / scheduler flags
attacked exactly this bucket at 8k.

**Predicted signal**: collective buckets shrink; step < 667.6 ms. PASS =
tok/s/chip ≥ +3% (step ≤ ~648 ms); loss parity (12.0914/12.0986/12.0947
±0.001 — flags are numerics-preserving per v026).

**Falsification**: step within noise of 667.6 ms ⇒ the flag wins are
shape-gated to long-seq (like SC-offload/HOST_OFFLOAD were gated the other
way, v009/v010) — port axis closes, batch axis next.

## Results

**0 training steps — killed at the 60-min wall budget while still compiling.**
Not a Tier-2 hang (CPU active, HLO modules still appearing: 230 at ~46 min
when the `[opt]`/`[ce]` setup markers printed, 348 at kill). Root cause: the
**unrolled 36-layer graph × the stack's scheduler passes**
(`--xla_latency_hiding_scheduler_rerun=2` +
`--xla_tpu_enable_all_experimental_scheduler_features=true`) — ~4× v052's
compile. At seq8192 this combination was never compiled: every flag-stack run
(v026+) used **scan** (3.4× compile reduction, v020). Submission/occupancy/
XPK mechanics all clean.
